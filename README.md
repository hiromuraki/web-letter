# Web Letter (网页贺卡)

这是一个基于 Go 语言和原生 Web 技术（HTML/CSS/JS）构建的轻量级互动网页贺卡。项目支持将前端静态资源和后端服务打包为单个可执行文件，方便部署。

## 1. 核心功能

- **单文件部署**：使用 Go 的 `//go:embed` 特性将静态资源打包进二进制文件。
- **交互动效**：包含纯 CSS 实现的 3D 信封展开动画，以及点击触发的星空和爱心交互。
- **音频支持**：支持背景音乐的淡入播放。
- **内存缓存机制**：信件文本缓存在后端内存中，并支持定期热重载（修改信件无需重启服务）。

## 2. 目录结构

```text
web-letter/
├── main.go                 # 后端主程序 (API 路由与静态资源托管)
├── core/
│   └── letter.go           # 信件相关逻辑
└── static/                 # 前端静态资源 (编译时打包)
    ├── index.html          # 主页面
    └── favicon.ico         # 网页图标
```

## 3. 配置说明

项目通过环境变量进行配置。如未设置环境变量，程序将默认读取当前目录下 `/data/` 文件夹中的内容。

| 环境变量名        | 说明             | 默认值             |
| :---------------- | :--------------- | :----------------- |
| `PORT`            | 服务运行端口     | `8000`             |
| `LETTER_PASSWORD` | 贺卡解锁口令     | `000000`           |
| `LETTER_FILE`     | 信件文本物理路径 | `/data/letter.txt` |
| `MUSIC_FILE`      | 背景音乐物理路径 | `/data/music.mp3`  |

**信件文本 (`letter`) 格式要求：**

```text
TITLE:这里写标题
SIGN:这里写署名
<注意：这里必须保留一个空行>
这是正文的第一段。

这是正文的第二段。

任何空行都将被忽略。
```

## 4. 运行与构建

### 4.1. 本地调试

```bash
# 启动服务
go run main.go

# 浏览器访问
http://localhost:8000
```

### 4.2. 编译部署

你可以将项目编译为单一文件，直接在服务器上运行。

```bash
# 交叉编译为 Linux 可执行文件
GOOS=linux GOARCH=amd64 go build -o web-letter main.go

# 部署时，仅需将 web-letter 可执行文件和 data 文件夹上传至服务器
./web-letter
```
